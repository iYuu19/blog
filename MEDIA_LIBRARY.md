# 媒体库说明

这个项目还是文件化博客，所以“媒体库”本质上不是数据库，而是统一管理你现在用到的图片目录和使用规则。

## 现在主要有两个目录

- `public/uploads`
  - 用途：正文截图、步骤说明图、文章中穿插的普通配图
  - 常见来源：在后台正文编辑器里直接粘贴截图、拖拽图片
- `public/contest-covers`
  - 用途：比赛封面、比赛索引卡、首页比赛入口的视觉图
  - 常见来源：手动挑一张更概括比赛主题的封面图

## 推荐用法

### 正文截图

- 最省事的方式就是边写边粘贴
- 截图会自动进入 `public/uploads`
- 如果一篇长文图片很多，后面可以再慢慢整理成同名子目录

### 比赛封面

- 建议单独放到 `public/contest-covers`
- 命名尽量和比赛标识一致
- 例如：`yellow-river-police-2026.svg`

## 命名建议

- 普通正文截图：可以先接受自动命名，优先保证写作流畅
- 长文章或专题文章：建议按文章 slug 建子目录
- 比赛封面：建议按比赛 slug 命名

## 压缩建议

- 截图优先 `webp` 或较轻的 `png`
- 封面图如果是图形类，`svg` 会更干净
- 图片太大时先压缩，再上传，前台加载会明显更顺

## 你最常遇到的场景

### 我想在正文里快速插图

直接在 `文章中心` 的正文里粘贴截图或拖图进去就可以。

### 我想给比赛页换封面

去文章里改 `coverImage`，图片尽量从 `public/contest-covers` 里选。

### 我想知道哪些目录是给什么用的

去后台的 `媒体库 -> 媒体目录与命名规则` 看说明卡，那里会列出每个目录的用途和建议格式。
